Работа с Git, GitHub:
# 1. Создать папку проекта (если ещё нет)
mkdir название_проекта

# 2. Перейти в папку
cd название_проекта

3. Инициализируйте Git
git init

4. Создайте файл .gitignore (если нужно)
# Для Python проекта
echo "__pycache__/" > .gitignore
echo "venv/" >> .gitignore
echo ".env" >> .gitignore

# Или через редактор
notepad .gitignore

5. Добавьте файлы в Git
# Добавить все файлы
git add .

# Или конкретные файлы
git add README.md main.py

6. Сделайте первый коммит
git commit -m "Initial commit"

Шаг 6: Создайте репозиторий на GitHub
 1. Зайдите на github.com и войдите

 2. Нажмите зеленую кнопку "New" или значок + → New repository

 3. Введите Repository name (например, my-new-project)

 4. Выберите Public или Private

 5. НЕ СТАВЬТЕ галочки "Add a README file", ".gitignore", "license"

 6. Нажмите "Create repository"

 Шаг 7: Свяжите локальный репозиторий с GitHub:
    # Добавьте удалённый репозиторий (URL возьмите со страницы GitHub)
    git remote add origin https://github.com/ВАШ_НИК/НАЗВАНИЕ_РЕПО.git

    # Проверьте
    git remote -v

Шаг 8: Отправьте код на GitHub:
# Если ветка называется master
git push -u origin master

# Если ветка называется main (современный стандарт)
git branch -M main  # переименовать master в main
git push -u origin main


Полный пример (все команды разом):
# 1. Создаём и переходим в проект
mkdir my-awesome-project
cd my-awesome-project

# 2. Инициализируем Git
git init

# 3. Создаём .gitignore (для Python)
echo "__pycache__/" > .gitignore
echo "venv/" >> .gitignore
echo ".env" >> .gitignore

# 4. Создаём README (опционально)
echo "# My Awesome Project" > README.md

# 5. Добавляем файлы и коммитим
git add .
git commit -m "Initial commit"

# 6. Переименовываем ветку в main
git branch -M main

# 7. Добавляем remote (ЗАМЕНИТЕ НА ВАШ URL!)
git remote add origin https://github.com/ВАШ_НИК/my-awesome-project.git

# 8. Отправляем на GitHub
git push -u origin main

Быстрая памятка (шпаргалка):
# Инициализация
git init

# Добавить все файлы
git add .

# Коммит
git commit -m "Первый коммит"

# Переименовать ветку (если нужно)
Посмотреть название ветки:
git branch
git branch -M main

# Добавить remote
git remote add origin https://github.com/пользователь/репозиторий.git

# Запушить
git push -u origin main

Если неправильно звязан Git и GitHub:
Способ 1: Удалить и добавить заново (рекомендуется):
# Посмотреть текущие remote
git remote -v

# Удалить неправильный remote
git remote remove origin

# Добавить правильный remote
git remote add origin https://github.com/Alex-top/booking_bot.git

# Проверить, что теперь всё верно
git remote -v


Способ 2: Изменить URL существующего remote:
# Изменить URL у origin
git remote set-url origin https://github.com/Alex-top/booking_bot.git

# Проверить
git remote -v
